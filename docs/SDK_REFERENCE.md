## 1. Authentication

GUAW uses **Sovereign API Keys** for all authenticated requests.
These keys follow the format: `sk_live_[hex_string]`.

### Usage

All requests must include the API Key in the custom header:

```http
X-API-Key: sk_live_your_secret_key
```

> **Security Note:** API Keys must **never** be embedded in client-side code (browsers, mobile apps). They must be kept in secure server-side environments.

> **Persistence Note:** We only store the SHA-256 hash of your key. If lost, it cannot be recovered. You must revoke and regenerate it via the **Guardian Console**.

## 2. Bio-Defense (Presence API)

The Presence API verifies if user interaction is biological or synthetic based on kinematic motor entropy.

### Endpoint

`POST /api/v1/presence/verify`

### Payload (Body)

```json
{
  "data": "kinematic_data_string",
  "context": {
    "module": "login_form",
    "difficulty": "high"
  }
}
```

### Responses

- **200 OK:** Verification complete.
  - **confidence:** A value between `0.0` (Bot) and `1.0` (Human).
  ```json
  {
    "status": "success",
    "verdict": "HUMAN",
    "confidence": 0.9982,
    "proofId": "prf_82kxa9...",
    "timestamp": "2026-02-12T14:40:00Z"
  }
  ```
- **429 Too Many Requests:** Quota exceeded.

## 4. Error Codes

GUAW uses standard HTTP status codes combined with specific machine-readable error reasons. All error responses include a machine-readable `error` field.

| Code  | Reason                    | Description                                     |
| ----- | ------------------------- | ----------------------------------------------- |
| `401` | `AUTHENTICATION_REQUIRED` | API Key missing in headers.                     |
| `401` | `INVALID_API_KEY`         | Provided key hash does not match.               |
| `429` | `QUOTA_EXCEEDED`          | Daily limit reached (High Performance Cluster). |
| `403` | `SCOPE_MISMATCH`          | Key does not have permission for this API.      |
